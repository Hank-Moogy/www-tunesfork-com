## Invite-link feature (new)

Design decision (Samori's): unknown emails are NOT emailed by Tunesfork.
The owner gets a personal link to share through their own channel. Existing
accounts are still added instantly + notified via Resend.

- **Migration `20260610140000_project_invites.sql`** (applied to remote):
  - `project_invites` table: email, `permission_level` enum, single-use
    `token` (hex, `extensions.gen_random_bytes`), 14-day `expires_at`,
    `accepted_by/accepted_at`. Partial unique index = one pending invite per
    (project, email); re-inviting refreshes expiry/permission instead.
  - RLS: owner-only management. No anon policy — token reads go through
    SECURITY DEFINER RPCs.
  - `create_project_invite(_project_id, _email, _permission)`: owner-only,
    anon revoked, 20 invites/day/user cap.
  - `get_project_by_share_token`, `get_versions_by_share_token`,
    `accept_share_invite` now resolve invite tokens **in addition to** the
    project-wide `share_token`. `accept_share_invite` grants the invite's
    permission level, marks the invite consumed, and does NOT consume it if
    the owner opens their own link.
- **Frontend:**
  - `src/App.tsx`: `/invite/:token` route renders the existing `SharePage`.
    The whole funnel is reused: invite hero → `/auth?invite=` →
    sessionStorage `tf_pending_invite` → `AuthRedirect` resumes to
    `/share/{token}` *before* onboarding → auto-accept → land in project →
    `notify-invite-accepted` emails the owner.
  - `src/pages/ProjectPage.tsx`: Add Collaborator dialog has two outcomes
    (existing user = added immediately; unknown = link mode with copy button
    and prefilled `mailto:`). Collaborators panel shows pending invites
    (owner-only) with copy-link and revoke.
  - New RPC/table calls use `(supabase as any)` like `promote_project_version`
    does — `src/integrations/supabase/types.ts` was not regenerated. Run
    `supabase gen types typescript --project-ref urrxrntdkmmmqqwaihfj` if you
    want typed calls.
